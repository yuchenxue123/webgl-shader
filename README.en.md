# GLSL Shader Playground

A browser-based GLSL fragment shader live-coding environment built with TypeScript, WebGL, and Vite.

## Quick Start

```bash
npm run dev
```

Open `http://localhost:5173/` — the first preset renders immediately. Edit the shader code in the editor and see changes auto-compile in real time.

## Features

- **Live compilation** — 400ms debounced recompile on input, or Ctrl+Enter for instant
- **Built-in uniforms** — `u_time`, `u_resolution`, `u_mouse` provided automatically
- **Preset library** — Example, Gradient, Plasma, Checkerboard, Raymarch Sphere, Frosted Glass
- **Auto uniform UI** — `uniform` declarations are parsed and a visual control panel is generated
- **Error reporting** — line numbers corrected to your source, canvas keeps rendering last working shader
- **localStorage** — editor content persists across sessions

## Uniform Annotations

Custom uniforms in your shader are automatically detected and surfaced as interactive controls in the sidebar. Use `//` comments on the uniform line to configure them:

```glsl
uniform float u_blur;         // range:0.05,1.0,0.01   min, max, step
uniform float u_speed;        // range:-5.0,5.0        no step = auto default
uniform vec3  u_color;        // color                  renders as a color picker
uniform vec3  u_position;     // (no annotation)        x/y/z sliders
```

| Annotation | Effect |
|---|---|
| `// range:min,max,step` | Set slider range and step |
| `// color` | Render vec3/vec4 as a color picker |

**Without annotations**, the defaults are:
- `float`/`int`: slider 0–1, step 0.01
- `vec2`/`vec3`/`vec4`: individual component sliders (x, y, z, w)
- `bool`: checkbox

## How It Works

1. **Vertex shader** is fixed — a full-screen quad in clip space
2. **Fragment shader** is user-authored, with a preamble that declares `u_time`, `u_resolution`, `u_mouse`
3. On each frame, the engine sets these uniforms and draws 6 vertices (two triangles)
4. Custom user uniforms are parsed from the source, cached by name, and set per-frame
5. On compile error, the previous working program stays active — the canvas never goes blank
